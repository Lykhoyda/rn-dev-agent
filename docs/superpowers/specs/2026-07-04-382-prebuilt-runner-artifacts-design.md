# Story 01 / GH #382 — Prebuilt runner artifacts (implementation design)

**Date:** 2026-07-04
**Story spec:** [`docs/stories/01-prebuilt-runner-artifacts.md`](../../stories/01-prebuilt-runner-artifacts.md)
**Issue:** [#382](https://github.com/Lykhoyda/rn-dev-agent/issues/382)
**Scope for this PR:** client foundation + workflow scaffold (see "Scope" below)

## Problem (recap)

First `device_snapshot action=open` cold-builds rn-fast-runner via `xcodebuild` (up to 6 min);
Android gradle-builds two APKs on first use. This is the worst possible first-run UX and
re-triggers on every runner-source-touching version bump. See the story spec for full context.

## Decisions locked (brainstorming, 2026-07-04)

1. **PR scope** — Client foundation + workflow scaffold. The client resolution module, wiring,
   version stamping/provenance, and doctor surface are fully implemented and unit-tested. The CI
   build jobs, manifest generation, and the release-asset download path are code-complete but
   **cannot be exercised in a local session** (need macOS/ubuntu runners + a live release). The
   download tier goes live only once a release ships assets.
2. **Trust model** — Committed manifest + release assets. `runner-manifest.json` is generated by CI
   and committed into the repo at release time, so it ships with the exact plugin version the user
   installed. The client reads it **offline** as the source of truth for expected SHA-256, then
   downloads zips from the GitHub Release for `v<version>` and verifies against the committed
   manifest. Offline cache-hit validation works with zero network. Single trust root tied to the
   installed version.

## Architecture

### New module: `scripts/cdp-bridge/src/runners/runner-artifacts.ts`

Public entry:

```ts
export type RunnerPlatform = 'ios' | 'android';
export type ArtifactProvenance = 'cache' | 'downloaded' | 'build-local';

export interface ResolvedIosArtifacts {
  provenance: ArtifactProvenance;
  derivedDataPath: string; // cache DD root when prebuilt; local build path when build-local
  note?: string;
}
export interface ResolvedAndroidArtifacts {
  provenance: ArtifactProvenance;
  appApk: string;   // cache path when prebuilt; gradle-output path when build-local
  testApk: string;
  note?: string;
}

export async function resolveRunnerArtifacts(
  platform: 'ios', version: string | null, deps?: ArtifactDeps,
): Promise<ResolvedIosArtifacts>;
export async function resolveRunnerArtifacts(
  platform: 'android', version: string | null, deps?: ArtifactDeps,
): Promise<ResolvedAndroidArtifacts>;
```

**Resolution order** (fail-open — the function never throws; any failure falls through to
`build-local` so a session is never blocked):

1. `RN_RUNNER_BUILD=local` (env) → `build-local` immediately.
2. `version === null` or committed `runner-manifest.json` has no entry for this version+platform
   → `build-local` (dev worktrees & pre-release builds fall through by design — visible via
   provenance, never silent).
3. **Cache hit** — `<cacheDir>/<version>/<platform>/` exists and every file named in the manifest
   is present with a matching SHA-256 → `cache`.
4. **Download** — `GET releases/download/v<version>/<assetName>` with: 60 s timeout; size cap =
   manifest `bytes` (+ small slack); SHA-256 verified **before** unzip; unzip with a
   path-traversal guard; atomically populate the cache dir → `downloaded`
   (+ `note: "downloading prebuilt runner (~N MB)"`).
5. Any download/verify/unzip failure (offline, 404, checksum mismatch, corrupt zip, traversal,
   oversize) → `build-local` (with a diagnostic `note`).

**Pure, individually unit-testable helpers** (no IO — mirrors the codebase's `resolveRunnerStartPlan`
/ `resolveAndroidInstallAction` style):

- `resolveArtifactDecision(input): 'cache' | 'download' | 'build-local'` — given
  `{ envOverride, manifestEntry, cacheState }`.
- `assertNoTraversal(entryNames: string[]): void` — rejects `..` segments and absolute paths.
- `verifyChecksums(expected, actualBySha): { ok: boolean; mismatched: string[] }`.
- `releaseAssetUrl(repo, version, assetName): string`.
- `cacheDirFor(version, platform, home, platformOS): string`.

**IO ports** live behind an injected `ArtifactDeps` (production impl `defaultArtifactDeps()`; tests
inject fakes — "fake fs + fake fetch"):

```ts
export interface ArtifactDeps {
  env: NodeJS.ProcessEnv;
  readManifest(): RunnerManifest | null;
  cacheRoot(): string;
  existsSync(p: string): boolean;
  sha256File(p: string): string;
  listFiles(dir: string): string[];
  fetchToFile(url: string, dest: string, opts: { timeoutMs: number; maxBytes: number }): Promise<void>;
  unzip(zipPath: string, destDir: string): void; // enforces assertNoTraversal
  mkdirp(p: string): void;
  rm(p: string): void;
}
```

No new npm dependency: the default `unzip` port shells out to `unzip` (lists entries with `-Z1`,
runs `assertNoTraversal`, then extracts with `-o -d`). `sha256File` uses `node:crypto`. `fetchToFile`
uses global `fetch` (Node ≥ 22) with an `AbortController` timeout and a byte cap while streaming.

### Manifest

Committed `runner-manifest.json` at the **plugin root** (read via a path resolution mirroring
`getPluginVersion()` in `runners/protocol.ts`):

```json
{
  "version": "0.62.3",
  "xcodeBuildVersion": "15.4",
  "assets": {
    "ios":     [{ "name": "rn-fast-runner-0.62.3-sim.zip", "sha256": "…", "bytes": 4200000 }],
    "android": [{ "name": "rn-android-runner-0.62.3.zip",  "sha256": "…", "bytes": 5100000 }]
  }
}
```

This PR commits an **empty** manifest (`{"version": null, "assets": {"ios": [], "android": []}}`) so
every current build resolves cleanly to `build-local`. CI overwrites it at release time.

### Integration (minimal — reuse the existing "warm" launch paths)

- **iOS** `rn-fast-runner-client.ts::startFastRunner`: call `resolveRunnerArtifacts('ios', …)` →
  use the returned `derivedDataPath` in `resolveRunnerStartPlan`. When prebuilt, the cache DD layout
  (`…/ios/Build/Products/{*-Runner.app, *.xctestrun}`) makes `hasBuiltTestProduct` true, so the plan
  is a single `test-without-building` — **no `build-for-testing`, no `xcodebuild build`**. When
  `build-local`, the existing local `derivedDataPathForRunner()` is used unchanged. Stamp
  `provenance` on `FastRunnerState`; surface `note` through the quiescence-announcement channel.
- **Android** `rn-android-runner-client.ts::ensureAndroidRunnerInstalled`: call
  `resolveRunnerArtifacts('android', …)` → install from the returned `{ appApk, testApk }`
  (cache paths when prebuilt) and **skip the gradle `assembleDebug`** when prebuilt. When
  `build-local`, gradle runs as today. Stamp `provenance` on `AndroidRunnerState`; set the
  pending-note.

### Version stamping + provenance

- Add `provenance?: 'prebuilt' | 'local'` to `FastRunnerState` / `AndroidRunnerState`
  (both already carry `runnerVersion`). Set from the resolution kind
  (`cache`/`downloaded` → `prebuilt`; `build-local` → `local`).
- The live 0.57.1-vs-0.57.3 cache-mirror mismatch class is **already** closed by the existing
  `version-skew` health gate (`classifyRunnerCompatibility`). Provenance is additive — for doctor
  diagnostics — and does not change reuse logic.

### Doctor / `cdp_status`

`tools/status.ts` + the `rn-dev-agent:doctor` skill report runner provenance:
`runner: prebuilt v0.62.3 (cache)` vs `local-built`, read from the runner state file.

### Release-workflow scaffold (code-complete, unverifiable locally)

- New `.github/workflows/runner-artifacts.yml` (or jobs appended to `release.yml`):
  - **iOS job** (macOS): `xcodebuild build-for-testing … CODE_SIGNING_ALLOWED=NO ONLY_ACTIVE_ARCH=NO`
    → zip `Debug-iphonesimulator/` + `*.xctestrun` as `rn-fast-runner-<version>-sim.zip`.
  - **Android job** (ubuntu): `./gradlew assembleDebug assembleDebugAndroidTest`
    → zip both APKs as `rn-android-runner-<version>.zip`.
  - **Publish**: `node scripts/build-runner-manifest.mts` computes SHA-256 + bytes, writes
    `runner-manifest.json`, creates GitHub Release `v<version>`, uploads both zips, commits the
    manifest. Gated on the existing version-sync check so artifact version == package version.
- `scripts/build-runner-manifest.mts` — manifest generator (pure enough to smoke-test locally with
  fixture files).

## Scope boundaries (this PR)

- **In:** `runner-artifacts.ts` + unit tests; iOS/Android wiring; provenance in state + doctor;
  committed empty manifest + manifest generator; workflow scaffold; README/CHANGELOG.
- **Out / deferred:** live download verification (needs a release with assets); nightly device-smoke
  integration (Story 06 Phase B); any change to the version-skew reuse gate.

## Test plan

Unit (`test/unit/gh-382-runner-artifacts.test.js`), all with fake fs + fake fetch:
`RN_RUNNER_BUILD=local` override · null version · manifest-missing-entry · cache-valid ·
cache-corrupt-checksum · cache-missing-file · download-ok · download-404 · offline/timeout ·
bad-checksum-after-download · corrupt-zip · traversal-attempt (`../`, absolute) · size-cap-exceeded.
Plus pure-helper tests for `resolveArtifactDecision`, `assertNoTraversal`, `verifyChecksums`,
`releaseAssetUrl`, `cacheDirFor`.

Manifest generator: fixture-file smoke (known bytes → known sha256).

Manual (recorded in PR body where possible): confirm a dev build resolves to `build-local`
(provenance `local`) and the runner still starts; confirm doctor prints `local-built`.

## Acceptance criteria (from story)

- Fresh machine + released version: first open reaches `RN_FAST_RUNNER_LISTENER_READY` in
  < 30 s + download; no xcodebuild (assert via absence of the cold-build marker). *(post-release)*
- Checksum mismatch / download failure → automatic `build-local` fallback with a clear `note`,
  never a hard failure. *(unit-tested)*
- `RN_RUNNER_BUILD=local` forces the old path. *(unit-tested)*
- Doctor shows `prebuilt v<X> (cache)` vs `local-built`. *(wired; local shows `local-built`)*

## Risks

- **xctestrun/Xcode compat** — build with oldest supported Xcode in CI; keep local-build fallback;
  record `xcodeBuildVersion` in the manifest for doctor diagnostics.
- **Release-asset availability for dev builds** — falls through to local build by design (provenance
  makes it visible).
- **Commit-back ordering** — CI must build → compute checksums → commit `runner-manifest.json` →
  publish assets for the same version. Validated on first real release, not locally.
