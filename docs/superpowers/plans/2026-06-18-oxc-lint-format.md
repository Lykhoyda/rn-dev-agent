# OXC (oxlint + oxfmt) Adoption Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adopt OXC's `oxlint` (linter) and `oxfmt` (formatter) as the lint + format layer for all hand-written code in the monorepo, format the existing code once, and enforce both via a blocking CI gate.

**Architecture:** Single root install + single root config for each tool (Approach A). `oxlint`/`oxfmt` are standalone Rust binaries that walk file paths and need no per-package `node_modules`/`tsconfig`, so one root `.oxlintrc.json` + one root `.oxfmtrc.json` cover the whole monorepo. Scoping is done entirely through `ignorePatterns`. The formatter covers code only — `.md`/`.mdx` prose docs are excluded (decision B2). A new `lint-format` CI job runs `oxlint` + `oxfmt --check` on push and PR.

**Tech Stack:** Node.js >= 22, npm workspaces, `oxlint` (stable, ~v1.70.x), `oxfmt` (beta), GitHub Actions, changesets.

## Global Constraints

_(Copied verbatim from the spec — every task's requirements implicitly include this section.)_

- **Node.js >= 22** (root `package.json` `engines` and CI already pin `node-version: 22`).
- **`oxfmt` is beta → pin its version EXACTLY** (`-E` / no caret). `oxlint` may use a caret.
- **CI actions are SHA-pinned** (Phase-134.5 convention). Reuse the repo's existing pins:
  - `actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd` (v6.0.2)
  - `actions/setup-node@48b55a011bda9f5d6aeb4c2d9c7362e8dae4041e` (v6.4.0)
- **Always excluded** from both tools: `node_modules/`, `dist/` (tracked `tsc` output), `WebDriverAgent/` (vendored Appium, both copies), `.claude/worktrees/`, `.astro/` cache, `.superpowers/`, `.remember/`.
- **Formatter (B2): code only.** `oxfmt` formats `.ts/.tsx/.js/.mjs/.cjs/.css`; `**/*.md` and `**/*.mdx` are excluded. `.astro` is unsupported by oxfmt and therefore untouched.
- **Invoke `oxfmt` over the repo root with NO explicit file-path args** — `ignorePatterns` is bypassed for paths passed explicitly on the CLI (oxc issue #17729).
- **No blanket rule/file disables.** Resolve lint findings; only narrow `// oxlint-disable-next-line <rule>` with a one-line reason is allowed.
- **A patch changeset is required** for `rn-dev-agent-cdp` (the `require-changeset` CI gate triggers on `scripts/cdp-bridge/src` changes).
- **`dist/` and the web bundle must stay in sync** — rebuild after formatting and commit only if they changed.

### Task 2: Add oxlint config + scripts; get the repo to lint-clean

**Files:**
- Create: `.oxlintrc.json` (root)
- Modify: `package.json` (root) — add `lint` / `lint:fix` scripts

**Interfaces:**
- Consumes: the `oxlint` binary from Task 1.
- Produces: `npm run lint` (exit 0 on a clean repo) — consumed by the CI gate in Task 4.

- [ ] **Step 1: Create `.oxlintrc.json`**

```json
{
  "$schema": "https://raw.githubusercontent.com/oxc-project/oxc/main/npm/oxlint/configuration_schema.json",
  "categories": {
    "correctness": "error"
  },
  "ignorePatterns": [
    "node_modules/",
    "dist/",
    "WebDriverAgent/",
    ".claude/worktrees/",
    ".astro/",
    ".superpowers/",
    ".remember/"
  ]
}
```

- [ ] **Step 2: Add the lint scripts to root `package.json`**

In the `"scripts"` block, add:
```json
"lint": "oxlint",
"lint:fix": "oxlint --fix",
```

- [ ] **Step 3: Run the linter to see the baseline findings**

Run: `npm run lint`
Expected: either exit 0 (clean) or a list of `correctness` errors. Record the count.

- [ ] **Step 4: Auto-fix what is safe, then re-run**

Run:
```bash
npm run lint:fix
npm run lint
```
Expected: fewer or zero errors after `--fix`.

- [ ] **Step 5: Hand-resolve any remaining findings**

For each remaining error, fix the code. Only if a finding is a genuine false positive, add a narrow inline suppression on the line above it with a reason, e.g.:
```ts
// oxlint-disable-next-line no-control-regex -- intentional: parsing ANSI escape codes
```
Do NOT disable a rule globally or for a whole file. Re-run `npm run lint` until it exits 0.

- [ ] **Step 6: Verify lint-clean**

Run: `npm run lint; echo "exit=$?"`
Expected: `exit=0`.

- [ ] **Step 7: Commit**

```bash
git add .oxlintrc.json package.json $(git diff --name-only)
git commit -m "chore(oxc): add oxlint config (correctness) and resolve findings

Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>"
```
(If Step 5 changed source files, they are included by the `git diff --name-only` expansion. Review `git status` before committing to confirm only intended files are staged.)

### Task 3: Add oxfmt config (B2) + scripts; run the one-time format pass

**Files:**
- Create: `.oxfmtrc.json` (root) — use the config-file name confirmed in Task 1 Step 3
- Modify: `package.json` (root) — add `format` / `format:check` scripts
- Modify (bulk, generated): every in-scope code file reformatted by oxfmt
- Modify (only if changed): `scripts/cdp-bridge/dist/**`, the committed web bundle

**Interfaces:**
- Consumes: the `oxfmt` binary from Task 1.
- Produces: `npm run format` (writes) and `npm run format:check` (exit 0 on a formatted repo) — `format:check` is consumed by the CI gate in Task 4.

- [ ] **Step 1: Create `.oxfmtrc.json` with the B2 ignore set**

```json
{
  "ignorePatterns": [
    "node_modules/",
    "dist/",
    "WebDriverAgent/",
    ".claude/worktrees/",
    ".astro/",
    ".superpowers/",
    ".remember/",
    "*.md",
    "*.mdx"
  ]
}
```
(`*.md` / `*.mdx` are the B2 exclusion — code only. `dist/` and `WebDriverAgent/` match at any depth via gitignore semantics.)

- [ ] **Step 2: Add the format scripts to root `package.json`**

In the `"scripts"` block, add:
```json
"format": "oxfmt",
"format:check": "oxfmt --check",
```
(Both invoke `oxfmt` with NO explicit path args so `ignorePatterns` is honored — see Global Constraints.)

- [ ] **Step 3: Dry-run the check to confirm scope BEFORE writing**

Run: `npm run format:check`
Expected: a list of files that WOULD be reformatted, plus stats. **Verify no `.md`/`.mdx` path appears** in the list and no `WebDriverAgent`/`dist`/`worktrees` path appears. If any do, fix `ignorePatterns` and re-run before proceeding.

- [ ] **Step 4: Run the one-time format pass (writes in place)**

Run: `npm run format`
Expected: oxfmt rewrites the in-scope code files (≈385 cdp-bridge + 4 root scripts + 1 web `.tsx` + docs-site `.ts`/`.css`).

- [ ] **Step 5: Verify the repo is now format-clean**

Run: `npm run format:check; echo "exit=$?"`
Expected: `exit=0`.

- [ ] **Step 6: Confirm B2 held — no prose docs were touched**

Run: `git status --porcelain | grep -E '\.(md|mdx)$' || echo "OK: no md/mdx changed"`
Expected: `OK: no md/mdx changed`.

- [ ] **Step 7: Confirm lint still passes on the reformatted code**

Run: `npm run lint; echo "exit=$?"`
Expected: `exit=0` (formatter runs last; verify it didn't reintroduce a lint error).

- [ ] **Step 8: Rebuild dist + web bundle; stage them only if they changed**

Run:
```bash
( cd scripts/cdp-bridge && npm run build )
bash scripts/check-web-bundle.sh
git status --porcelain scripts/cdp-bridge/dist scripts/cdp-bridge/src/observability/web
```
Expected: `npm run build` succeeds; `check-web-bundle.sh` exits 0 (bundle still fresh). If `dist/` shows changes, they are legitimate rebuilt output — stage them in the commit below. If nothing changed, that's expected (`tsc` output is whitespace-independent).

- [ ] **Step 9: Commit the format pass as one self-contained commit**

```bash
git add .oxfmtrc.json package.json
git add -A
git commit -m "style(oxc): format codebase with oxfmt (code only; prose docs excluded)

One-time big-bang format pass. .md/.mdx excluded per design decision B2.
Rebuilt dist where source changed.

Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>"
```
(Review `git status` before `git add -A` to confirm only reformatted code + config + any rebuilt `dist` are staged.)

### Task 4: Wire the blocking CI gate

**Files:**
- Modify: `.github/workflows/ci.yml` — add a `lint-format` job

**Interfaces:**
- Consumes: `npm run lint` (Task 2) and `npm run format:check` (Task 3).
- Produces: a required CI job that blocks merges on lint errors or format drift.

- [ ] **Step 1: Add the `lint-format` job**

In `.github/workflows/ci.yml`, add a new job under `jobs:` (sibling to `test:`):
```yaml
  lint-format:
    name: Lint & format
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd  # v6.0.2 — Phase 134.5 SHA pin
      - uses: actions/setup-node@48b55a011bda9f5d6aeb4c2d9c7362e8dae4041e  # v6.4.0 — Phase 134.5 SHA pin
        with:
          node-version: 22
          cache: npm
          cache-dependency-path: package-lock.json
      - run: npm ci
      - name: oxlint
        run: npm run lint
      - name: oxfmt check
        run: npm run format:check
```

- [ ] **Step 2: Validate the workflow YAML parses**

Run: `node -e "require('js-yaml')" 2>/dev/null && node -e "const y=require('js-yaml');y.load(require('fs').readFileSync('.github/workflows/ci.yml','utf8'));console.log('yaml ok')" || python3 -c "import yaml,sys; yaml.safe_load(open('.github/workflows/ci.yml')); print('yaml ok')"`
Expected: `yaml ok`.

- [ ] **Step 3: Simulate the CI job locally (clean checkout semantics)**

Run:
```bash
npm ci
npm run lint
npm run format:check
```
Expected: all three exit 0.

- [ ] **Step 4: Commit**

