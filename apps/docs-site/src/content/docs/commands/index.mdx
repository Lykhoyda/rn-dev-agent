## Host invocation

All fifteen workflows are first-class on both hosts:

```text
Claude: /rn-dev-agent:<workflow> [arguments]
Codex:  $rn-dev-agent:<workflow> [request text]
```

Codex also has ten domain skills, for exactly 25 native skills. Its workflow
skills are explicit-only and best-effort `source-command-*` migration is
disabled. The primary workflow is `rn-feature-dev`; the same names below apply
to both hosts.

| Command | When to use |
|---------|-------------|
| [`rn-feature-dev`](/rn-dev-agent/commands/rn-feature-dev/) | Build a complete feature with the 8-phase pipeline |
| [`test-feature`](/rn-dev-agent/commands/test-feature/) | Test an already-implemented feature end-to-end |
| [`build-and-test`](/rn-dev-agent/commands/build-and-test/) | Build the app first, then test |
| [`debug-screen`](/rn-dev-agent/commands/debug-screen/) | Diagnose a broken screen — gather evidence, apply fix |
| [`observe`](/rn-dev-agent/commands/observe/) | Watch the agent live in a web UI — timeline, device mirror, state panels, browser-run actions |
